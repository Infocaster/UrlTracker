UrlTracker

The Url Tracker is used to manage URLs within Umbraco. It automatically tracks 404 occurences and URL changes, for instance when a node is renamed or moved, and makes sure the old URL will redirect to the new location. This is great for SEO and great for people visiting your website via this old URL. Search engines will update the indexed URL and people won't visit the old, broken URL. You can also create your own redirects, based on a simple URL or using a Regex pattern. You can redirect to an existing node or a manually entered URL. This is great for migrating existing indexed URLs to your new website!

Settings

The Url Tracker supports a few settings, which can be configured in the appSettings section of your web.config. Below you'll find these settings, with the setting name, type, default value and current value (top right corner) mentioned

UrlTracker:disabled

{{vm.settings.IsDisabled}}
boolean (false)

Set to true to disable the HTTP Module of the Url Tracker, so it won't redirect requests

urlTracker:enableLogging

{{vm.settings.EnableLogging}}
boolean (false)

Set to true to enable logging debug information of the Url Tracker. Uses umbraco's built-in logging mechanism with LogType set to 'debug'.

urlTracker:trackingDisabled

{{vm.settings.TrackingDisabled}}
boolean (false)

Set to true to disable tracking URL changes

urlTracker:notFoundTrackingDisabled

{{vm.settings.IsNotFoundTrackingDisabled}}
boolean (false)

Set to true to disable tracking not found (404) requests

urlTracker:appendPortNumber

{{vm.settings.AppendPortNumber}}
boolean (true)

Set to false to disable appending a port number to redirect URLs
